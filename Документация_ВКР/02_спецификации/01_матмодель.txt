ГЛАВА «МАТЕМАТИЧЕСКАЯ МОДЕЛЬ И ОБОСНОВАНИЕ ВЫБОРА МЕТОДОВ»

Примечание к версии. Документ переработан в соответствии с финальными
решениями (журнал решений, раздел «Решения по функционалу 7–13»).
Ключевые изменения относительно предыдущей редакции: расчёт
энергетической потребности по формуле Миффлина — Сан Жеора и
мультипликативная модель TDEE с пятиуровневой шкалой активности
исключены; энергетическая норма берётся непосредственно из таблиц
МР 2.3.1.0253-21 по полу, возрасту и группе коэффициента физической
активности (четыре группы со значениями 1,4; 1,6; 1,9; 2,2; для лиц
старше трудоспособного возраста 1,7); бюджет и стоимостная оптимизация
исключены полностью; стоимость рациона не входит ни в постановку задачи,
ни в штрафную функцию; добавлены механизм закрепления блюд и механизм
переноса недобора микронутриентов на следующую неделю.


1. РАСЧЁТ ЦЕЛЕВЫХ ПОКАЗАТЕЛЕЙ ПИТАНИЯ

1.1. Энергетическая потребность

Модель — табличная, по нормативу. Целевая суточная энергетическая
потребность E_norm определяется не расчётной формулой, а выбором
значения из таблиц норм физиологических потребностей в энергии
МР 2.3.1.0253-21 по трём дискретным признакам пользователя: пол g,
возрастная группа age_grp и группа коэффициента физической активности
kfa_grp.

    E_norm = TableMR_energy(g, age_grp, kfa_grp).

Возрастные группы взрослого населения по МР 2.3.1.0253-21: 18–29,
30–44, 45–64, 65–74, 75 лет и старше. Группы коэффициента физической
активности (КФА) для трудоспособного населения — четыре, со значениями
1,4; 1,6; 1,9 и 2,2; для лиц старше трудоспособного возраста
используется значение 1,7.

Обоснование выбора. Существуют два принципиально разных подхода
к определению индивидуальной энергетической потребности: расчётный
(оценка основного обмена по антропометрическим уравнениям с последующим
умножением на коэффициент физической активности) и нормативно-табличный
(прямое использование утверждённых норм для возрастно-половых
и активностных групп населения).

                                 индивида        в РФ
    --------------------------------------------------------------------
    Расчётный (Миффлин — Сан     выше по         не является
    Жеор, Харрис — Бенедикт,     антропометрии   утверждённой нормой
    Кэтч — МакАрдл) × PAL        конкретного     для РФ; зарубежные
                                 человека        выборки

Выбран нормативно-табличный подход. Решающим аргументом является
нормативный статус: МР 2.3.1.0253-21 — действующий документ
государственного санитарно-эпидемиологического нормирования Российской
Федерации, прямо устанавливающий нормы физиологической потребности
в энергии для групп населения. Использование утверждённой нормы делает
расчёт целевого показателя верифицируемым по первоисточнику и устраняет
необходимость обоснования выбора одной антропометрической формулы среди
конкурирующих. Расчётные формулы основного обмена (включая
Миффлина — Сан Жеора) в модели не используются; рост и масса тела
сохраняются как антропометрические данные профиля пользователя и
применяются для контроля динамики массы тела, но в определение
энергетической нормы не входят.

Группа КФА определяется не прямым выбором пользователя из списка, а по
результатам мини-анкеты (дерево решений) о характере дневной активности,
регулярности и интенсивности физической нагрузки; отнесение к одной из
четырёх групп выполняется по решающим правилам анкеты. Обоснование
анкетного определения КФА приведено в разделе 1.4.


1.2. Поправка на цель

Модель: E* = E_norm · k_goal,    k_goal ∈ {0,85; 1,00; 1,15}.

Значение k_goal = 0,85 соответствует цели снижения массы тела
(энергетический дефицит 15 %), k_goal = 1,00 — поддержанию массы тела,
k_goal = 1,15 — набору массы тела (профицит 15 %). Величина E* является
итоговым дневным целевым числом энергетической потребности, к которому
модель стремится при формировании рациона на каждый день.

Обоснование выбора ±15 %. Альтернативы:
  — Фиксированное смещение −500 ккал/день (правило «3500 ккал = 0,5 кг
    жира») — не учитывает исходную норму: для низкой нормы это может
    составлять чрезмерный относительный дефицит, ведущий к катаболизму
    мышечной ткани.
  — Динамическая модель Холла (NIH Body Weight Planner) — решение
    дифференциального уравнения с адаптацией метаболизма, требует
    численного интегрирования и калибровки, избыточно для массового
    сегмента.
  — Процентная поправка ±15 % — рекомендация ISSN (International Society
    of Sports Nutrition, 2014), обеспечивающая безопасный темп изменения
    массы тела (0,5–1 % в неделю).

Процентная модель масштабируется пропорционально нормативному значению
и безопасна для всех групп без дополнительных параметров. Пользователю
разрешается ручная корректировка целевых показателей, но строго в
пределах ±15 % от нормы по МР 2.3.1.0253-21; при попытке выйти за этот
коридор значение фиксируется на его границе (box-constraint-проекция на
допустимый отрезок).


1.3. Распределение макронутриентов

Модель — нормативные доли калорийности по МР 2.3.1.0253-21
с пересчётом в граммы по энергетическим коэффициентам Этуотера
(белок и углеводы — 4 ккал/г, жир — 9 ккал/г):

    P* = E* · p_p / 4,    F* = E* · p_f / 9,    C* = E* · p_c / 4,
    p_p + p_f + p_c = 1,

где доли p_p, p_f, p_c берутся из МР 2.3.1.0253-21 для соответствующей
группы: доля жира в суточной калорийности взрослого трудоспособного
населения принята равной 30 %; доля белка снижается с ростом физической
нагрузки от 14 до 12 % калорийности (для лиц старше 65 лет — 14 %);
доля углеводов составляет 56–58 %.

Обоснование. Сами доли распределения калорийности взяты из
нормативного документа предметной области, что согласовано с подходом
раздела 1.1. Энергетические коэффициенты Этуотера используются только
как переводной множитель «калории → граммы»: это международный стандарт
(CODEX Alimentarius, FAO 2003), и справочные данные о химическом составе
продуктов (справочник под редакцией Скурихина и Тутельяна) приводятся
в согласованных с ним единицах, что обеспечивает единство единиц при
расчёте состава блюд.


1.4. Определение группы физической активности

Модель — дерево решений (мини-анкета). Группа КФА kfa_grp ∈ {I, II,
III, IV} определяется по ответам пользователя на вопросы анкеты
о характере обычной дневной активности, регулярности отдельной
физической нагрузки и её интенсивности. Каждой группе соответствует
нормативное значение КФА из МР 2.3.1.0253-21.

Обоснование анкетного подхода. Альтернатива — прямой выбор пользователем
уровня активности из списка категорий. Недостаток прямого выбора
зафиксирован как структурный недостаток процесса (раздел 3): уровень
активности определяется на основе субъективной самооценки, и
распространённой ошибкой является отнесение работы, выполняемой стоя без
значительной нагрузки, к категории сидячей, что занижает расчётную
потребность в энергии. Мини-анкета как дерево решений снижает влияние
самооценки: пользователь отвечает на конкретные вопросы о деятельности,
а отнесение к группе выполняется по фиксированным правилам, что повышает
воспроизводимость определения группы КФА по сравнению со свободным
выбором категории.


2. РАСПРЕДЕЛЕНИЕ ЦЕЛЕЙ ПО ПРИЁМАМ ПИЩИ

Модель — фиксированные доли с ренормализацией:

    share_s' = share_s / Σ share_i    (по активным приёмам пищи i),
    share = (0,25; 0,35; 0,30; 0,10).

Дневное целевое число E* распределяется по активным приёмам пищи
пропорционально нормализованным долям; аналогично распределяются
целевые P*, F*, C*.

Обоснование. Альтернативы:
  — Равномерное деление 1/n — противоречит физиологии: основная часть
    энергии должна приходиться на активную фазу дня.
  — Оптимизация долей под хронотип пользователя (циркадная модель) —
    требует дополнительных входных данных и не обоснована для массового
    сегмента.
  — Пропорции 25/35/30/10 — медианный паттерн, соответствующий
    рекомендациям по режиму питания.

Линейная нормализация сохраняет соотношение активных приёмов пищи при
отключении неактивных; математически это проекция на симплекс
{x : Σ x_i = 1, x_i ≥ 0}, сохраняющая структуру распределения.

Корректировка при закреплённых блюдах. Если пользователь закрепил блюда
в части приёмов пищи (раздел 5), их фактические показатели не подбираются
заново, а вычитаются из дневного целевого числа: остаточные целевые
показатели E*_rem = E* − Σ E(закреплённые), P*_rem, F*_rem, C*_rem
распределяются по описанному правилу только между свободными приёмами
пищи. Если остаток по какому-либо показателю становится отрицательным
или свободных приёмов нет, план помечается как не удовлетворяющий
коридору, и инициируется процедура информирования пользователя
(раздел 4.4).


3. ФОРМАЛИЗАЦИЯ ЗАДАЧИ ГЕНЕРАЦИИ ПЛАНА

3.1. Постановка задачи

Пусть:
  — D = {d_1, ..., d_N} — множество допустимых блюд после фильтрации
    ограничений (тип питания, аллергии, индивидуально исключаемые
    продукты, список исключённых блюд пользователя, соответствие типу
    приёма пищи);
  — T = {1, ..., |T|} — дни планирования (|T| = 7);
  — M — множество приёмов пищи (завтрак, обед, ужин, перекус);
  — M_fix ⊆ M — приёмы пищи с закреплёнными пользователем блюдами;
  — M_free = M \ M_fix — свободные приёмы пищи;
  — x_{d,t,m} ∈ {0,1} — выбрано ли блюдо d в день t в приём пищи m;
  — α_{d,t,m} ∈ [α_min, α_max] — множитель числа порций.

Задача:
    min  Σ_{t ∈ T} Σ_{m ∈ M_free} F(d*(t,m), t, m)
     x,α

при ограничениях:
    Σ_{d ∈ D} x_{d,t,m} = 1    для всех t, m ∈ M_free
        (ровно одно блюдо в каждый свободный приём пищи);
    x_{d,t,m} = 1 фиксировано для m ∈ M_fix
        (закреплённые блюда заданы пользователем и не подбираются).

Стоимостное ограничение в постановке отсутствует. Стоимость рациона
не оптимизируется и не входит ни в ограничения, ни в целевую функцию;
ориентировочная стоимостная оценка списка покупок выполняется отдельно
и постфактум (раздел 6).


3.2. Целевая функция (функция-штраф)

    F(d, t, m) =
        w_1 · |E_d − E*_m| / E*_m
      + w_2 · (1/3) · ( |P_d − P*_m| / P*_m
                      + |F_d − F*_m| / F*_m
                      + |C_d − C*_m| / C*_m )
      + w_3 · 1[ τ(d, t) ≤ k ],

где E_d, P_d, F_d, C_d — энергетическая ценность и содержание белков,
жиров, углеводов выбранного блюда с учётом множителя порции;
E*_m, P*_m, F*_m, C*_m — целевые показатели приёма пищи m (с учётом
ренормализации и остатка при закреплённых блюдах, раздел 2);
1[·] — индикатор; τ(d, t) — число дней с момента последнего включения
блюда d в план; k — горизонт контроля повторяемости.

Стоимостное слагаемое и слагаемое предпочтений («лайков») в функции
отсутствуют: стоимость не оптимизируется (раздел 6), положительные
предпочтения реализуются механизмами закрепления и замены блюд, а не
весом в F (журнал решений, решения по функционалу №6 и №11).

Задача имеет природу многокритериальной оптимизации (энергетическая

Почему взвешенная сумма:
  1. Теорема Жоффриона (Geoffrion, 1968): любая точка Парето-фронта
     выпуклой задачи достижима подбором весов в линейной свёртке.
     Допустимая область задачи по непрерывной релаксации α близка
     к выпуклой, поэтому потеря качества мала.
  2. Интерпретируемость: веса w_i имеют смысл «важности» критерия.
  3. Вычислительная сложность: оценка одного кандидата — O(1).


3.4. Обоснование относительной нормализации отклонений

Используется относительная L1-норма |x − x*| / x*, а не абсолютная
|x − x*| и не квадратичная (x − x*)^2.

Почему не абсолютная: критерии имеют разные масштабы (энергия — сотни
ккал, белок — десятки граммов). Без нормализации доминировал бы
крупномасштабный признак.

Почему L1, а не L2:
  — L2 квадратично штрафует большие отклонения; здесь однократное
    отклонение по одному приёму пищи не критично, если день в целом
    сбалансирован.
  — L1 робастна к выбросам и не «выбраковывает» все блюда с отклонением
    выше порога, что сохраняет разнообразие на дискретном множестве
    блюд.
  — L1 интерпретируема в процентах и понятна как «отклонение на N %».

Почему деление на target, а не на dish: цель x*_m — константа приёма
пищи, а не блюда, поэтому нормализатор не зависит от выбранного d, что
сохраняет монотонность по отклонению.


3.5. Обоснование линейной штрафной структуры

Альтернатива — мультипликативная F = Π f_i^{w_i} или Чебышёвская
F = max_i w_i f_i.

  — Мультипликативная недопустима: при идеальном попадании f_i = 0
    всё произведение обнуляется, игнорируя остальные критерии.
  — Чебышёвская теряет чувствительность к «мелким» критериям.
  — Линейная — устойчива, непрерывна, легко калибруется.

4.2. Фаза 1 — жадный подбор

На каждом свободном приёме пищи (t, m ∈ M_free) выбирается блюдо
с минимальным значением F независимо от других приёмов пищи. Это
локально оптимальное решение в каждом подпространстве (t, m).

Когда жадность обоснована:
  — Приёмы пищи квазинезависимы по целевым показателям: каждый имеет
    собственные E*_m, P*_m, F*_m, C*_m, полученные нормализацией
    (с учётом остатка при закреплённых блюдах) — критерий аддитивен
    по приёмам пищи.
  — Контроль повторяемости учитывается через состояние τ(d, t) —
    память о последних k днях, что превращает жадность в алгоритм
    с динамическим обновлением состояния (аналог k-tabu).

Гарантии качества. Для задач с аддитивно-сепарабельным по подзадачам
критерием жадная стратегия даёт решение, близкое к оптимуму
поприёмно; теоретический ориентир — оценка (1 − 1/e) для субмодулярных
функций (Немхаузер — Вулси — Фишер, 1978), причём аддитивно-сепарабельная
структура F ставит задачу в более выгодные условия.

4.3. Фаза 2 — локальный поиск для балансировки по КБЖУ

Жадный подбор минимизирует штраф каждого приёма пищи по отдельности,
но не контролирует суммарные дневные и недельные показатели: сумма
поприёмно оптимальных блюд может систематически отклоняться от дневного
целевого числа. Для устранения этого отклонения применяется вторая фаза.

    Фаза 1: жадное решение, минимизирующее поприёмный штраф.
    Фаза 2: hill-climbing в пространстве замен — итеративно заменяется
            блюдо, дающее наибольший вклад в суммарное дневное
            отклонение по энергии и БЖУ, на блюдо из допустимого
            множества, уменьшающее это отклонение, до тех пор пока
            суммарные дневные показатели не войдут в допустимый коридор
            или пока улучшение не прекратится.

Обоснование двухфазности:
  1. Жадная фаза обеспечивает поприёмное качество, но задача
     согласования суммарных показателей с дневной целью является
     отдельной и в общем виде NP-трудной; двухфазная схема
     (построение допустимого решения, затем его улучшение) —
     классический приём теории приближённых алгоритмов.
  2. Порядок обхода «от блюда с наибольшим вкладом в отклонение» —
     эвристика максимального вклада в нарушение целевого показателя,
     минимизирующая число итераций.

Замена в фазе 2 выполняется только в пределах того же приёма пищи
и того же допустимого множества блюд; закреплённые приёмы пищи (M_fix)
в локальном поиске не участвуют. Дополнительно масштабируется число
порций (раздел 5) как непрерывный механизм подгонки.

4.4. Контроль соответствия и информирование пользователя

После фазы 2 выполняется проверка соответствия плана:
  — суммарные дневные показатели энергии и БЖУ — в пределах коридора
    относительно дневного целевого числа;
  — нормы по микронутриентам — по итогам недели (раздел 7).

Если соответствие не достигается (например, из-за большого числа
закреплённых блюд, не оставляющих свободным приёмам пищи достаточного
ресурса для компенсации), план не отбрасывается автоматически, а
пользователю выводится предупреждение с указанием показателя, по
которому план не укладывается, и предлагаются варианты действий: снять
часть закреплений, принять план с явно зафиксированным отклонением либо
переформировать план. Окончательное решение принимает пользователь;
алгоритм не выполняет действие, изменяющее введённые пользователем
ограничения, без его подтверждения.

4.5. Ограничение на рост штрафа при замене

    F(new) ≤ c · F(old),    c = 1,8.

Обоснование. Без ограничения локальный поиск ради сведения суммарного
баланса мог бы заменить сбалансированное по приёму пищи блюдо на блюдо,
сильно ухудшающее поприёмный штраф. Константа c = 1,8 — компромиссный
коэффициент, допускающий умеренное локальное ухудшение ради улучшения
суммарного баланса и не допускающий вырожденных замен. Формально это
trust-region-ограничение: замена принимается только если она не слишком
далеко от текущего решения в метрике F.


5. МАСШТАБИРОВАНИЕ ПОРЦИИ

Модель:
    α*(d, m) = clip( E*_m / E_d,  α_min,  α_max ),
    [α_min, α_max] = [0,7; 2,0].

Обоснование формулы. Задача одномерной оптимизации
    min_α |α · E_d − E*_m|
имеет аналитическое решение α = E*_m / E_d (точка обращения модуля
в ноль), что следует из условия Каруша — Куна — Таккера для выпуклой
задачи.

Обоснование ограничений [0,7; 2,0]. Без ограничений алгоритм мог бы
предложить нереалистичное число порций:
  — α < 0,7 — порция становится следовой, не обеспечивает насыщения;
  — α > 2,0 — нарушает физиологический объём приёма пищи и форматы
    продаваемых продуктов.

Формально — проекция безусловного оптимума на допустимый отрезок
(box-constraint-projection).


6. ОРИЕНТИРОВОЧНАЯ ОЦЕНКА СТОИМОСТИ

Стоимость рациона не является ни критерием оптимизации, ни ограничением
задачи генерации плана. После того как план и список покупок
сформированы, пользователю предоставляется опциональная ориентировочная
стоимостная оценка списка покупок.

Модель. Пусть I — множество позиций агрегированного списка покупок,
g_i — требуемое количество позиции i. Для одного выбранного пользователем
магазина по данным внешнего ценового агрегатора формируется ориентировочная
стоимостная вилка:

    cost_min = Σ_{i ∈ I} g_i · p_i^min,
    cost_max = Σ_{i ∈ I} g_i · p_i^max,

где p_i^min и p_i^max — минимальная и максимальная цена позиции i среди
сопоставленных товаров выбранного магазина. Результат предъявляется
пользователю как диапазон «от cost_min до cost_max».

Обоснование. Минимизация стоимости, распределение покупок по нескольким
магазинам и проверка фактического наличия товаров исключены из модели
как решения, не относящиеся к задаче формирования сбалансированного
рациона и не предусмотренные финальной постановкой. Оценка носит
исключительно справочный характер: пользователь выбирает один магазин
либо пропускает этот этап, и в последнем случае стоимостная оценка не
формируется. Тем самым алгоритм не оптимизирует за пользователя
стоимостную метрику, а лишь предоставляет ориентир по выбранному им
магазину.


7. УЧЁТ МИКРОНУТРИЕНТОВ — ПЕРЕНОС НЕДОБОРА

7.1. Обоснование вынесения микронутриентов из функции F

Замечание о включении микронутриентов в основную итоговую формулу
учтено по существу, но реализовано не их включением в F. В перечне
МР 2.3.1.0253-21 нормируется порядка 29 микронутриентов. Прямое
включение их в штрафную функцию превращает свёртку из четырёх критериев
(энергия, белки, жиры, углеводы) приблизительно в 33 критерия, каждому
из которых требуется весовой коэффициент. Обоснование почти трёх
десятков весов нормативным источником невозможно, а произвольные веса
делают модель неустойчивой и недоказуемой. Кроме того, множество блюд,
одновременно удовлетворяющих всем нормативам микронутриентов в пределах
одного дня, практически пусто, что приводит к систематической
неразрешимости задачи. Поэтому сбалансированность по микронутриентам
обеспечивается не критерием в F, а проверкой по факту сформированного
плана и компенсацией недобора на следующей неделе. F остаётся
устойчивой и доказуемой, а цель замечания (контроль сбалансированности
по микронутриентам) достигается.

7.2. Модель переноса недобора

Горизонт контроля микронутриентов — неделя; «фактическим» потреблением
принимается запланированное (отметки пользователя о реально съеденном
не используются).

По итогам недели для каждого нормируемого микронутриента i
рассчитывается среднесуточное запланированное потребление N_plan_i
(взвешенная сумма содержаний по продуктам всех блюд плана, делённая на
число дней) и сравнивается с нормой N_norm_i для возрастно-половой
группы пользователя по МР 2.3.1.0253-21:

    Δ_i = max(0, N_norm_i − N_plan_i)    (недельный недобор,
                                          в пересчёте на сутки).

При формировании плана на следующую неделю целевое значение
микронутриента i повышается:

    N_target_i = min( N_norm_i + Δ_i / 7,  UL_i ),

где UL_i — верхний допустимый уровень потребления. Источник UL_i —
МР 2.3.1.1915-04; для микронутриентов, не охваченных этим документом,
используются сводные таблицы верхних допустимых уровней EFSA.

Недобор не накапливается между итерациями: на каждой неделе учитывается
только недобор предыдущей недели (Δ_i рассчитывается заново). Целевые
значения N_target_i применяются не как слагаемые в F, а как мягкий
приоритет при жадном подборе: при прочих равных значениях F
предпочтение отдаётся блюду, сокращающему наибольший относительный
недобор по микронутриентам; основной критерий (энергия и БЖУ) при этом
не нарушается, поскольку приоритет применяется только среди блюд,
близких по F.

7.3. Отсутствие пользовательского отчёта

Значения недобора и компенсации хранятся во внутренней структуре данных
и в пользовательский интерфейс не выводятся: отчёт по микронутриентам,
перечень дефицитов и их визуализация пользователю не предоставляются
(журнал решений, решение по функционалу №13).
