# Smoke-набор для тестирования ядра

Минимально достаточный CSV-набор для проверки работоспособности
ядра до завершения полной выгрузки справочника.

## Состав

- **products.csv** — 20 базовых продуктов из 8 категорий: молочные, мясо,
  рыба, яйца, овощи, фрукты, крупы, хлеб, жиры, орехи, бобовые. БЖУ-цифры —
  из справочника Скурихин/Тутельян (издание 2007).
- **micronutrients.csv** — 6 микронутриентов (ca, fe, mg, k, zn, vit_d).
  UL — из МР 2.3.1.1915-04 / EFSA.
- **micronutrient_norms.csv** — нормы для 2 возрастных групп (18-29, 30-44)
  и обоих полов. Источник — МР 2.3.1.0253-21.
- **product_micronutrients.csv** — содержание 6 нутриентов в 20 продуктах.
  Источник — Скурихин/Тутельян 2007.
- **energy_norms.csv** — 16 строк (2 возраста × 2 пола × 4 группы КФА).
  Источник — МР 2.3.1.0253-21.

## Запуск

```sh
export DATABASE_DSN="postgres://Kirill@localhost:5432/nutrition_dev"
loader load-all --data-dir data/smoke
```

## Зачем

- Smoke-тест ядра (можно гонять алгоритм без полной выгрузки ion.ru).
- Воспроизводимый набор для unit-тестов на стыке репозитория и алгоритма.
- Контрольный пример для раздела 13 ВКР (при необходимости расширить).

## Не использовать в продакшне

Это подмножество. Полная выгрузка делается отдельно
(ручная в `data/full/` либо автопарсер ion.ru — follow-up задача).
