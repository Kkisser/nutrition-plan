# core/ — ядро формирования плана питания (Go)

Реализация ядра по `../МАТМОДЕЛЬ.txt`, `../АЛГОРИТМ_блок-схема.svg`,
`../СХЕМА_БД.md`, `../КОНТРАКТ_API.md`.

## Требования

- Go 1.23+
- PostgreSQL 14+ (миграции — `db/migrations`, инструмент — `goose`)
- Redis (кэш — позже, когда дойдём до интеграции)
- `price-service` запущен на `http://localhost:8085` (опционально, для оценки стоимости)

## Планируемая структура

```
core/
├── cmd/
│   └── server/             — HTTP-сервер ядра, точка входа
├── internal/
│   ├── domain/             — типы профиля, целей, плана, блюд, продуктов
│   ├── targets/            — расчёт целевых КБЖУ (energy_norms × поправка цели)
│   ├── catalog/            — фильтрация каталога блюд по профилю
│   ├── planner/            — двухфазный алгоритм (greedy + hill-climbing)
│   ├── micronutrients/     — расчёт недобора и переноса на след. неделю
│   ├── shopping/           — агрегация списка покупок
│   ├── repository/         — интерфейсы + postgres-реализация
│   ├── pricing/            — HTTP-клиент к price-service
│   └── api/                — HTTP-хендлеры по КОНТРАКТ_API.md §2
└── db/
    ├── migrations/         — goose .sql
    └── README.md
```

Структура расширяется по мере написания кода — пустые папки не создаются заранее.

## Запуск (план, на момент когда ядро будет реализовано)

```sh
# миграции
goose -dir db/migrations postgres "$DATABASE_DSN" up

# сервер
go run ./cmd/server
```

## Что вне core/

- `../price-service/` — отдельный stateless Go-сервис, считает вилку цены.
- Python-загрузчик справочника Скурихина-Тутельяна — будет в `../loader/` (П1).
- Фронтенд React/TS — будет в `../web/` (позже).
